# セキュリティポリシー

## サポート対象バージョン

C3 は基本的に**最新リリース**（[PyPI](https://pypi.org/project/claude-code-conductor/) の最新版）を
サポート対象とします。修正は新しいバージョンとしてリリースされます。

## 脆弱性の報告方法

脆弱性を発見した場合は、**公開 Issue や Pull Request には書かないでください**。

GitHub の **Private vulnerability reporting**（非公開の脆弱性報告）をご利用ください:

1. リポジトリの **Security** タブを開く
2. **Report a vulnerability** から報告フォームを送信

報告には、再現手順・影響範囲・想定される攻撃シナリオを含めていただけると助かります。
受領後、できる範囲で速やかに確認し、修正方針と公開時期について連絡します。

## 脅威モデルに関する補足

C3 は **ローカルで動作する開発支援ツール**であり、`.claude/` 配下の設定・スキル・
エージェント定義は基本的に「利用者本人が管理する信頼済み入力」という前提で設計されています。
この前提のもとで「許容例外」と判断している項目もあります。報告いただいた内容がこの脅威モデルの
範囲内かどうかも含めて確認します（範囲内であっても、防御を強化できる場合は対応を検討します）。
