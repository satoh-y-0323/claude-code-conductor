# /init-session コマンド

セッションを初期化し、前回の作業状態を復元する。
セッション開始時に手動で実行する。

## 実行手順

### Step 1: 前回セッションを読み込む

Glob で `.claude/memory/sessions/*.tmp` を検索し、ファイル名（YYYYMMDD）が最大のものを Read する。
ファイルが存在しない場合は「前回セッションなし」として Step 3 へ進む。

### Step 2: 昇格候補パターンを確認する

`.claude/memory/patterns.json` を Read する（存在する場合）。
`promotion_candidate: true` のパターンがあれば、Step 3 のサマリに含める。

### Step 3: サマリを表示する

以下の形式で表示する:

---
## セッション再開 ({今日の日付})

### 前回セッション ({前回の日付})
**残タスク:**
{残タスクリスト。完了済み[x]は省略する}

**前回うまくいったこと:**
{成功アプローチの要約}

**注意事項（前回失敗したこと）:**
{失敗アプローチの要約。なければ省略}

### 昇格候補パターン
{promotion_candidate: true のパターン一覧。なければ「なし」}

---

続きから作業しますか？それとも新しいタスクを開始しますか？

### Step 4: ユーザーの回答に応じて分岐する

**「続きから」の場合:**
残タスクの内容を確認してそのまま作業に入る。

**「新しいタスク」の場合:**
以下を確認する:

```
どんな作業を予定していますか？
  [small]  小さい修正・調査・質問
  [medium] 機能追加・バグ修正
  [large]  新機能設計・大規模変更
```

| 選択 | 案内 |
|---|---|
| small | 直接指示してください。エージェントは不要です |
| medium | `/agent-developer` から始めましょう |
| large | フルワークフロー（`/agent-interviewer` から）を使いましょう |

---

## session ファイルの更新ルール（LLM が守ること）

session ファイルはタスク完了のたびに更新する。まとめて最後に書かない。

**更新するタイミングと内容:**

| タイミング | 更新内容 |
|---|---|
| タスク完了時 | 残タスクの該当行を `[x]` にする |
| 良いアプローチを発見したとき | `## うまくいったアプローチ` に追記 |
| 失敗・ハマったとき | `## 試みたが失敗したアプローチ` に追記 |
| パターンを発見したとき | JSON ブロックの `patterns` 配列に追記 |
| 新しいタスクが発生したとき | `## 残タスク` に追記 |

**パターンの書き方（JSON ブロック内）:**
```json
"patterns": [
  {
    "id": "一意なID（英数字・アンダースコア）",
    "description": "どんな状況でどう対処するかを1文で"
  }
]
```

`stop.py` が session 終了時に `patterns` を読み取り、`patterns.json` の信用度を自動更新する。
