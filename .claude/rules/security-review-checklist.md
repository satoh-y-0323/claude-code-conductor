# Security Review Checklist

各項目には `[SR-XX-NNN]` 形式の checklist_id を付与している。
レビュー時に指摘する場合はこの ID を併記すること（review-hint 機能の照合キー）。

**該当 ID がない指摘**は `[SR-NEW]` を使う。無理やり近い既存 ID にマッピングしないこと（review-hint の照合精度が落ち、チェックリストの成長機会も失われるため）。`[SR-NEW]` 指摘は本チェックリストへの追加候補として扱う。

## 必須チェック項目（OWASP Top 10 準拠・全作成物共通）

### インジェクション
- [ ] [SR-INJ-001] SQLインジェクションの対策がされているか（プレースホルダ・パラメータバインド使用等）
- [ ] [SR-INJ-002] OSコマンド・インジェクションの対策がされているか（シェル呼び出し禁止・引数エスケープ）
- [ ] [SR-INJ-003] XSS（クロスサイトスクリプティング）の対策がされているか（出力エスケープ・CSP）
- [ ] [SR-INJ-004] LDAPインジェクションの対策がされているか（LDAP使用時）
- [ ] [SR-INJ-005] XMLインジェクション・XXEの対策がされているか（XML処理時）

### 認証・認可
- [ ] [SR-AUTH-001] 認証が適切に実装されているか
- [ ] [SR-AUTH-002] 認可チェックが全エンドポイントに実装されているか
- [ ] [SR-AUTH-003] セッションID固定化攻撃の対策がされているか（ログイン後にセッションIDを再生成しているか）
- [ ] [SR-AUTH-004] セッション管理が適切か（タイムアウト・Secure属性・HttpOnly属性）
- [ ] [SR-AUTH-005] セッションハイジャック対策がされているか（Secure/HttpOnly Cookie・トークン検証）
- [ ] [SR-AUTH-006] ブルートフォース・辞書・リスト攻撃の対策がされているか（アカウントロックアウト・レートリミット・CAPTCHA）
- [ ] [SR-AUTH-007] リプレイ攻撃の対策がされているか（ノンス・タイムスタンプ・ワンタイムトークン検証）
- [ ] [SR-AUTH-008] パスワードが適切にハッシュ化されているか（bcrypt・Argon2等）

### JWT・トークン管理
- [ ] [SR-JWT-001] JWT の署名アルゴリズムが適切か（`alg: none` を許可していないか・HS256/RS256等を明示指定しているか）
- [ ] [SR-JWT-002] JWT の有効期限（exp）が設定されているか
- [ ] [SR-JWT-003] JWT の秘密鍵・公開鍵が適切に管理されているか（ハードコードされていないか）
- [ ] [SR-JWT-004] リフレッシュトークンの無効化（ローテーション・ブラックリスト）が実装されているか
- [ ] [SR-JWT-005] OAuth2 / OIDC を使用する場合、state パラメータで CSRF 対策がされているか

### 秘密情報
- [ ] [SR-K-001] APIキー・パスワード・トークンがハードコードされていないか
- [ ] [SR-K-002] 秘密情報が .gitignore 対象ファイルで管理されているか
- [ ] [SR-K-003] ログに秘密情報が出力されていないか

### 入力値バリデーション
- [ ] [SR-V-001] ユーザー入力が全てバリデーションされているか
- [ ] [SR-V-002] ディレクトリトラバーサルの対策がされているか（パスのサニタイズ・正規化・絶対パス検証）
- [ ] [SR-V-003] ファイルアップロードの検証が実装されているか（拡張子・MIMEタイプ・サイズ）
- [ ] [SR-V-004] リダイレクト先URLの検証がされているか（オープンリダイレクト対策）

### SSRF（Server-Side Request Forgery）
- [ ] [SR-S-001] サーバーから外部URLへリクエストを行う箇所で、送信先URLがユーザー入力から生成されていないか
- [ ] [SR-S-002] 内部ネットワーク・クラウドメタデータエンドポイント（169.254.169.254等）へのアクセスをブロックしているか
- [ ] [SR-S-003] URLのホスト名をホワイトリストで制限しているか（または DNS リバインディング対策をしているか）
- [ ] [SR-S-004] HTTPリダイレクトのフォローを無効化または制限しているか

### メモリ安全性
- [ ] [SR-MEM-001] バッファオーバフロー攻撃の対策がされているか（境界チェック・安全なAPI使用・メモリ安全な言語の選択）

### 通信の安全性
- [ ] [SR-T-001] HTTPS/TLSが強制されているか（スニファ・電波傍受対策）
- [ ] [SR-T-002] HSTSが設定されているか
- [ ] [SR-T-003] 証明書の検証が正しく実装されているか（自己署名証明書の無効化等）

### 依存関係
- [ ] [SR-DEP-001] 依存ライブラリに既知の脆弱性がないか（npm audit / pip audit 等）
- [ ] [SR-DEP-002] 不要な権限・スコープを持つライブラリがないか

## 推奨チェック項目（全作成物共通）
- [ ] [SR-R-001] エラーメッセージに内部情報（スタックトレース・DB構造等）が含まれていないか
- [ ] [SR-R-002] セキュリティヘッダーが適切に設定されているか（X-Frame-Options・X-Content-Type-Options等）
- [ ] [SR-R-003] CORS の設定が適切か（`Access-Control-Allow-Origin: *` の多用・不必要なオリジン許可がないか）
- [ ] [SR-R-004] セキュリティイベント（認証失敗・認可エラー・不正入力等）が適切にログ記録されているか（OWASP A09 対策）

## 作成物別チェック項目

### Webアプリケーション向け
- [ ] [SR-WA-001] CSRF（クロスサイトリクエストフォージェリ）の対策がされているか（CSRFトークン・SameSite Cookie）
- [ ] [SR-WA-002] フォームジャッキング攻撃の対策がされているか（CSP・SRI によるスクリプト改ざん検知）
- [ ] [SR-WA-003] MITB（Man-in-the-Browser）攻撃の対策がされているか（CSP・サブリソース完全性 SRI の設定）

### AI機能を持つシステム向け
- [ ] [SR-AI-001] プロンプトインジェクション攻撃の対策がされているか（入力サニタイズ・出力検証・システムプロンプトの分離）

### メール送信機能向け
- [ ] [SR-EM-001] スパムメール対策がされているか（送信レートリミット・CAPTCHA・SPF/DKIM/DMARC設定）
- [ ] [SR-EM-002] メールヘッダーインジェクションの対策がされているか

### 公開Webサービス向け
- [ ] [SR-PUB-001] DDoS・F5アタック（大量リクエスト）への対策がされているか（アプリケーション層のレートリミット・リクエスト数制限）

### 外部APIやDNSに依存するシステム向け
- [ ] [SR-DNS-001] DNSキャッシュポイズニングの影響を軽減できているか（HTTPS証明書検証により名前解決が汚染されても通信先を検証できるか）
